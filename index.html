Testes Influunt-API

  • Toda funcionalidade DEVE estar acompanhada de seu respectivo teste (JUnit).
  • Não commitar código com testes falhando. Para rodar os testes do projeto: activator test
  • Durante o desenvolvimento de uma feature, checar no Codacy a qualidade do código do seu branch (cada pull request é automaticamente analizado).
  • Para analisar um branch antes de abrir o pull request:
    • Dar um push no branch p/ o repositório.
    • Habilitar a análise de código para o branch aqui.

Os testes estão localizados em influunt-api/test

Code Review

  • Todo pull request deverá passar pelo processo de code review.
  • Você deve fazer o code review dos pull requests dos outros membros da equipe.
  • O merge do pull request será feito se, e somente se:
    • Todos os problemas encontrados durante o code review forem resolvidos.
    • O Travis deu resultado positivo na compilação e resultado dos testes.
    • O Codacy deu resultado positivo para a qualidade do código.
